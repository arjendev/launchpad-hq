## Decisions Captured

1. **Tunnel Lifecycle**: Temporary (auto-delete on close) — simplest for P2, no state management
2. **Authentication**: Pre-login (user runs `devtunnel user login` beforehand) — keeps launchpad simple
3. **URL Extraction**: Regex parsing on stdout — devtunnel outputs text, not JSON
4. **Implementation Pattern**: Adapt self-daemon spawner — proven lifecycle management
5. **Error Handling**: Clear status codes (`running`, `not_running`, `not_available`, `auth_failed`, `error`) — actionable for dashboard
6. **CLI Integration**: Simple `--tunnel` flag in CLI — minimal change, consistent pattern
7. **Auth for QR Code (P2+)**: Token-based with short expiration — instant mobile access, secure
