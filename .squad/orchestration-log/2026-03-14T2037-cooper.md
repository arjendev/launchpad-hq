## Key Decisions Made

| Decision | Choice | Reasoning |
|----------|--------|-----------|
| Tunnel Lifecycle | Temporary (auto-delete on close) | Simplest for P2; no state persistence needed |
| Authentication | Pre-Login (user responsibility) | Keeps launchpad simple; clear error messaging |
| URL Extraction | Regex parsing on stdout | devtunnel outputs to stdout, not JSON; stable |
| Implementation Pattern | Adapt Self-Daemon Spawner | Proven pattern: stdio piping, exit handling, graceful shutdown |
| Error Handling | Clear status codes | `/api/tunnel` returns `{ status, url?, error? }` with specific codes |
| CLI Integration | Simple flag pass-through | `--tunnel` flag in CLI, `TUNNEL_ENABLED` env var for server |
